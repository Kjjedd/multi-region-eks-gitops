# 트러블슈팅 기록

> 멀티 리전 EKS 구축 과정에서 발생한 주요 이슈와 해결 방법

## 핵심 이슈 요약

| 이슈 | 원인 | 해결 방법 |
|------|------|-----------|
| RDS Access Denied | boarduser 미생성 | mysql client Pod로 직접 계정 생성 |
| Deployment 구조 오류 | serviceAccountName 위치 오류 | spec 레벨로 이동 |
| SecretProviderClass 타입 오류 | secretObjects 위치 오류 | spec 직속으로 이동 |
| DB_HOST localhost 접속 실패 | Secrets Manager에 host 키 없음 | ConfigMap에 DB_HOST 추가 |
| 노드 Public 서브넷 배치 | eksctl 서브넷 우선순위 | --node-private-networking 사용 |
| TGW ping 실패 | Security Group 미설정 | 상대방 CIDR 인바운드 추가 |
| Cross-account 접근 불가 | IAM Role 미설정 | Cross-account Role + assume |
| SecretProviderClass ARN 불일치 | 리전별 ARN 미패치 | Kustomize overlay에서 패치 |
| CSI Driver 토큰 오류 | tokenRequests 미설정 | CSIDriver에 tokenRequests 추가 |
| 오사카/도쿄 RDS 접근 불가 | Private DNS 조회 불가 | 로컬 MySQL Pod 사용 |
| Ingress Progressing | ALB Controller 미설치 | 각 클러스터에 Controller 설치 |



## 주요 이슈 상세

### 1. RDS Access Denied

**문제**: RDS 생성 시 애플리케이션용 계정이 자동 생성되지 않음

**해결**:
```bash
kubectl run -it --rm mysql-client --image=mysql:8.0 --restart=Never -n board-app-eks \
  -- mysql -h <RDS엔드포인트> -u admin -p'<패스워드>'
```

```sql
CREATE USER 'boarduser'@'%' IDENTIFIED BY 'boardpass';
GRANT ALL PRIVILEGES ON boarddb.* TO 'boarduser'@'%';
FLUSH PRIVILEGES;
```

### 2. Kubernetes 리소스 구조 오류

**Deployment 구조**:
```yaml
spec:
  serviceAccountName: secret-sa   # containers와 같은 레벨
  volumes:
    - name: secrets-store
  containers:
    - name: board-app-container
      volumeMounts:               # containers 안에
        - name: secrets-store
```

**SecretProviderClass 구조**:
```yaml
spec:
  provider: aws
  parameters:
    objects: |
      ...
  secretObjects:    # spec 바로 아래
    - secretName: board-app-secret
```

### 3. TGW 멀티 리전 연결

**Security Group 설정 필수**:
```
EKS 노드 ClusterSharedNodeSecurityGroup에 상대방 CIDR 인바운드 추가:
- 서울: 10.20.0.0/16 (오사카), 10.30.0.0/16 (도쿄)
- 오사카: 192.168.0.0/16 (서울)
- 도쿄: 192.168.0.0/16 (서울)
```

### 4. Cross-account EKS 접근

**해결**:
```bash
aws eks update-kubeconfig \
  --region ap-northeast-3 \
  --name osaka-staging-eks \
  --role-arn arn:aws:iam::<대상계정ID>:role/TeamA-EKS-Access \
  --alias osaka-staging-eks
```

### 5. Kustomize Overlay 리전별 패치

**오사카 overlay에서 SecretProviderClass ARN 패치**:
```yaml
- patch: |-
    - op: replace
      path: /spec/parameters/objects
      value: |
        - objectName: 'arn:aws:secretsmanager:ap-northeast-3:...'
  target:
    kind: SecretProviderClass
    name: aws-secrets
```

### 6. CSI Driver TokenRequests

**필수 설정**:
```bash
kubectl edit csidriver secrets-store.csi.k8s.io
```

```yaml
tokenRequests:
  - audience: sts.amazonaws.com
```

## 디버깅 명령어

```bash
# Pod 상태 확인
kubectl get pods -n board-app-eks
kubectl describe pod <pod-name> -n board-app-eks
kubectl logs <pod-name> -n board-app-eks --previous

# Secret 확인
kubectl get secret -n board-app-eks
kubectl describe secret <secret-name> -n board-app-eks

# Ingress 확인
kubectl get ingress -n board-app-eks
kubectl describe ingress <ingress-name> -n board-app-eks
```
