# EKS 클러스터 구축 가이드

> 멀티 리전 EKS 클러스터 구축을 위한 핵심 절차 요약

## 핵심 구성 요소

- **EKS 클러스터**: eksctl을 통한 IaC 기반 프로비저닝
- **네트워킹**: Private Subnet 배치, TGW Peering 지원
- **보안**: IRSA, Secrets Manager 연동, OIDC Provider
- **인그레스**: AWS Load Balancer Controller (ALB)
- **스토리지**: EBS CSI Driver, Secrets Store CSI Driver



## 클러스터 생성 (eksctl)

## 클러스터 생성 (eksctl)

```bash
eksctl create cluster \
  --vpc-private-subnets <프라이빗서브넷1>,<프라이빗서브넷2> \
  --name $CLUSTER_NAME \
  --region $REGION \
  --version 1.31 \
  --nodegroup-name <노드그룹이름> \
  --node-type t3.medium \
  --nodes 1 \
  --nodes-min 1 \
  --nodes-max 2 \
  --node-private-networking
```

**핵심 옵션**:
- `--node-private-networking`: 노드를 Private Subnet에 배치 (보안)
- `--vpc-private-subnets`: 워커 노드가 배치될 서브넷 지정

## OIDC Provider 등록

## OIDC Provider 등록

IRSA (IAM Roles for Service Accounts) 사용을 위한 필수 설정:

```bash
eksctl utils associate-iam-oidc-provider --cluster $CLUSTER_NAME --approve
```

## AWS Load Balancer Controller 설치

## AWS Load Balancer Controller 설치

ALB Ingress를 위한 필수 컨트롤러

### 1. IAM Policy 생성

```bash
aws iam create-policy \
  --policy-name AWSLoadBalancerControllerIAMPolicy \
  --policy-document file://infra/iam_policy.json
```

### 2. IRSA 생성

```bash
eksctl create iamserviceaccount \
  --cluster $CLUSTER_NAME \
  --namespace kube-system \
  --name aws-load-balancer-controller \
  --role-name AmazonEKSLoadBalancerControllerRole \
  --attach-policy-arn arn:aws:iam::$ACCOUNT_ID:policy/AWSLoadBalancerControllerIAMPolicy \
  --approve
```

### 3. Helm 설치

```bash
helm repo add eks https://aws.github.io/eks-charts
helm install aws-load-balancer-controller eks/aws-load-balancer-controller \
  -n kube-system \
  --set clusterName=$CLUSTER_NAME \
  --set serviceAccount.create=false \
  --set serviceAccount.name=aws-load-balancer-controller \
  --set region=$REGION \
  --set vpcId=$VPC_ID
```

## Secrets Manager 연동

## Secrets Manager 연동

### 1. ServiceAccount 생성

```bash
eksctl create iamserviceaccount \
  --name secret-sa \
  --namespace board-app-eks \
  --cluster $CLUSTER_NAME \
  --attach-policy-arn arn:aws:iam::aws:policy/SecretsManagerReadWrite \
  --approve
```

### 2. CSI Driver 설치

```bash
helm repo add secrets-store-csi-driver https://kubernetes-sigs.github.io/secrets-store-csi-driver/charts
helm install -n kube-system csi-secrets-store \
  --set syncSecret.enabled=true \
  --set enableSecretRotation=true \
  secrets-store-csi-driver/secrets-store-csi-driver

# TokenRequests 설정
kubectl edit csidriver secrets-store.csi.k8s.io
# 추가: tokenRequests: [{ audience: sts.amazonaws.com }]

# AWS Provider 설치
kubectl apply -f https://raw.githubusercontent.com/aws/secrets-store-csi-driver-provider-aws/main/deployment/aws-provider-installer.yaml
```

## 배포 및 확인

### 리소스 배포 순서

```bash
kubectl apply -f manifests/board-app/base/Namespace.yaml
kubectl apply -f manifests/board-app/base/Configmap.yaml
kubectl apply -f manifests/board-app/base/SecretProviderClass.yaml
kubectl apply -f manifests/board-app/base/Deployment.yaml
kubectl apply -f manifests/board-app/base/Service.yaml
kubectl apply -f manifests/board-app/base/Ingress.yaml
```

### 상태 확인

```bash
# Pod 상태
kubectl get pods -n board-app-eks -w

# Ingress ALB 주소
kubectl get ingress -n board-app-eks

# 로그 확인
kubectl logs -n board-app-eks deploy/board-app
```
